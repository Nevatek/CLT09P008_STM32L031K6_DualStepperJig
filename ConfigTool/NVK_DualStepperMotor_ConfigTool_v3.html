📡 Binary Packet Protocol

All communication uses fixed-framing binary packets over UART. Every packet is framed with a Start-of-Packet byte, a command byte, a data-length byte, up to 255 data bytes, and a 2-byte CRC-16/CCITT checksum.

■ Packet Frame
▶ Structure
SOP      0xAA          1 byte  — Start of packet
CMD      0x01–0x04     1 byte  — Command ID
DATALEN  0x00–0xFF     1 byte  — Payload length
DATA     …             0–255 bytes
CRC1     high byte     CRC-16/CCITT of CMD+DATALEN+DATA
CRC2     low byte      CRC-16/CCITT of CMD+DATALEN+DATA
↑ Transmit — App → Device

Sent when Send Configuration is pressed. Both motors stop immediately on receipt.

▶ CMD 0x01 — CFG (23 bytes payload, little-endian)
Byte 00   M1 Steps        uint16 LE (lo)
Byte 01   M1 Steps        uint16 LE (hi)
Byte 02   M1 Dir          0x00=CW  0x01=CCW
Byte 03   M1 Home         0x00=off 0x01=on
Byte 04   M1 Mode         0x00=AUTO 0x01=MANUAL
Byte 05   M1 µStep        log2(divisor): 0=full … 8=1/256
Byte 06–07M1 Freq         uint16 LE Hz
Byte 08–09M1 StepAngle    uint16 LE deg×10: 9=0.9° 18=1.8° 25=2.5° 27=2.7° 36=3.6° 72=7.2° 75=7.5°
Byte 10–11M1 StrokeDelay  uint16 LE ms (100–5000, step 100)
Byte 12   M2 Steps        uint16 LE (lo)
Byte 13   M2 Steps        uint16 LE (hi)
Byte 14   M2 Dir          0x00=CW  0x01=CCW
Byte 15   M2 Home         0x00=off 0x01=on
Byte 16   M2 Mode         0x00=AUTO 0x01=MANUAL
Byte 17   M2 µStep        log2(divisor): 0=full … 8=1/256
Byte 18–19M2 Freq         uint16 LE Hz
Byte 20–21M2 StepAngle    uint16 LE deg×10: 9=0.9° 18=1.8° 25=2.5° 27=2.7° 36=3.6° 72=7.2° 75=7.5°
Byte 22–23M2 StrokeDelay  uint16 LE ms (100–5000, step 100)
Byte 24   Sync            0x00=independent 0x01=lockstep
↓ Receive — Device → App

Device responds with a framed binary packet. CMD byte identifies the response type.

✓ CMD 0x02 — ACK (0 bytes data)
AA 02 00 CRC1 CRC2   Config accepted
✕ CMD 0x03 — ERR (1 byte data)
AA 03 01 ERR_CODE CRC1 CRC2
  0x01 = invalid steps
  0x02 = invalid frequency
  0x03 = CRC mismatch
  0xFF = generic error
↓ Receive — Cycle Count

After ACK, device sends EEPROM cycle counts for both motors.

↻ CMD 0x04 — CYC (8 bytes data, little-endian)
Byte 00–03 M1 cycles  uint32 LE
Byte 04–07 M2 cycles  uint32 LE
↻ Cycle Counter Reset

Sent when ↺ Reset Count is pressed on a cycle card. Device zeroes the selected motor's EEPROM cycle counter and replies with ACK.

▶ CMD 0x05 — RST (1 byte payload)
Byte 00   Motor mask  0x01=Motor 1  0x02=Motor 2  0x03=Both
